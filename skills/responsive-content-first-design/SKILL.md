# 響應式設計 + 內容優先系統

**技能 ID**：`responsive-content-first-design`  
**版本**：v1.0  
**最後更新**：2026-06-15  
**維護者**：好創團隊

## 📋 技能概述

使用 clamp() 動態計算實現完美響應式設計，優先考慮內容而非設計，確保手機端和電腦端都能提供優秀的用戶體驗。

### 核心能力
- 使用 clamp() 動態計算實現完美響應式
- 手機端 + 電腦端同時優化（不是「先電腦再手機」）
- 內容結構決定視覺設計（而不是反向）
- 建立可複用的設計系統（色彩、排版、間距）

### 適用場景
- ✅ 內部工具開發（需要手機端訪問）
- ✅ 客戶交付物（報告、提案在手機上也要清晰）
- ✅ 新人培訓網站（新人可能在手機上查看）
- ✅ 公開網站（需要支持所有設備）
- ✅ 移動應用（需要適配各種屏幕尺寸）

```

```

- 無陰影：none

### Phase 2：使用 clamp() 實現動態計算

#### Step 2.1：理解 clamp() 語法
```
clamp(最小值, 推薦值, 最大值)

示例：
font-size: clamp(14px, 2.5vw, 18px)
- 最小 14px（手機端）
- 推薦 2.5vw（根據視口寬度）
- 最大 18px（電腦端）

計算方式：
- 視口寬度 375px：2.5% × 375 = 9.375px → 最小值 14px
- 視口寬度 768px：2.5% × 768 = 19.2px → 最大值 18px
- 視口寬度 1200px：2.5% × 1200 = 30px → 最大值 18px
```

#### Step 2.2：應用 clamp() 到各個元素
```
【標題】
h1 {
  font-size: clamp(28px, 6vw, 48px);
  margin-bottom: clamp(16px, 3vw, 24px);
}

【正文】
p {
  font-size: clamp(14px, 2.5vw, 18px);
  line-height: 1.6;
  margin-bottom: clamp(12px, 2vw, 20px);
}

【按鈕】
button {
  font-size: clamp(12px, 2vw, 16px);
  padding: clamp(8px, 1.5vw, 12px) clamp(12px, 2vw, 16px);
  border-radius: 4px;
}

【卡片】
.card {
  padding: clamp(12px, 2vw, 20px);
  border-radius: 8px;
  box-shadow: 0 1px 2px rgba(0,0,0,0.05);
}

【容器】
.container {
  max-width: 1280px;
  margin: 0 auto;
  padding: 0 clamp(12px, 3vw, 32px);
}
```

### Phase 3：建立響應式設計原則

#### Step 3.1：移動優先原則
```
設計流程：
1. 先設計手機版本（375px）
2. 再擴展到平板版本（768px）
3. 最後擴展到電腦版本（1280px+）

不要：
❌ 先設計電腦版本，再「縮小」到手機版本
✅ 先設計手機版本，再「擴展」到電腦版本
```

#### Step 3.2：內容優先原則
```
設計決策：
1. 內容決定設計，不是設計決定內容
2. 不要為了設計而刪除或縮小內容
3. 不要為了設計而改變內容的含義
4. 設計應該服務於內容，而不是相反

示例：
❌ 「這段文字太長了，我們刪掉一半」
✅ 「這段文字很重要，我們調整排版讓它更易讀」
```

#### Step 3.3：流動性原則
```
設計方法：
1. 使用 flexbox 和 grid
2. 避免固定寬度（除非必要）
3. 讓內容自然流動
4. 使用 clamp() 動態計算

示例：
❌ width: 300px;
✅ width: clamp(250px, 90%, 400px);
```

#### Step 3.4：可讀性原則
```
設計標準：
1. 文字大小足夠（最小 14px）
2. 行距足夠（最小 1.4）
3. 顏色對比足夠（WCAG AA 標準）
4. 點擊區域足夠（最小 44px × 44px）

檢查清單：
☐ 文字大小是否足夠
☐ 行距是否足夠
☐ 顏色對比是否足夠
☐ 按鈕是否容易點擊
☐ 表單是否容易填寫
```

## 💡 最佳實踐

### 1. 使用 clamp() 而不是媒體查詢
```css
/* ❌ 不好：需要多個媒體查詢 */
.title { font-size: 32px; }
@media (max-width: 768px) {
  .title { font-size: 24px; }
}
@media (max-width: 480px) {
  .title { font-size: 20px; }
}

/* ✅ 好：一行代碼 */
.title { font-size: clamp(20px, 5vw, 32px); }
```

### 2. 建立設計系統
- 定義所有顏色、排版、間距
- 文檔化所有設計決策
- 確保整個團隊遵循同一套系統

### 3. 優先考慮內容
- 內容決定設計，不是反向
- 不要為了設計而刪除內容
- 設計應該服務於內容

## ⚠️ 常見陷阱

### 陷阱 1：過度使用媒體查詢
❌ 為每個屏幕尺寸寫一個媒體查詢  
✅ 使用 clamp() 動態計算

### 陷阱 2：先設計電腦版本
❌ 先設計 1920px 版本，再「縮小」到手機  
✅ 先設計 375px 版本，再「擴展」到電腦

### 陷阱 3：為了設計而改變內容
❌ 「這段文字太長了，我們刪掉一半」  
✅ 「調整排版讓文字更易讀」

### 陷阱 4：忽視可讀性
❌ 「文字很小，但看起來很精緻」  
✅ 「確保文字大小足夠，顏色對比足夠」
