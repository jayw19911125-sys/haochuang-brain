# 內容準確性驗證流程

**技能 ID**：`content-accuracy-verification`  
**版本**：v1.0  
**最後更新**：2026-06-15  
**維護者**：Dennis COO

---

## 📋 技能概述

確保所有內容（文檔、教材、報告、提案）都基於真實數據，沒有胡編亂造、過時信息或矛盾數據。

### 核心能力
- 從多個數據源提取真實資訊（Notion、Google Drive、GitHub、Monday.com）
- 自動驗證所有連結、數據、文案的準確性
- 建立「單一真實來源」（SSOT）機制
- 防止質量問題在發布後被發現

### 適用場景
- ✅ 新人培訓內容（FAQ、教材、文檔）
- ✅ 客戶交付物（報告、提案、分析文件）
- ✅ 內部流程文檔（SOP、工具清單、責任矩陣）
- ✅ 對外宣傳內容（案例、定價、服務介紹）

## 🔄 執行流程

### Phase 1：建立驗證清單

#### Step 1.1：識別內容類型
```
內容類型分類：
├── 教材類（新人培訓、FAQ、教程）
├── 報告類（分析報告、提案、案例研究）
├── 流程類（SOP、工作流程、責任矩陣）
└── 宣傳類（服務介紹、定價、案例）
```

#### Step 1.2：確定數據源
```
主要數據源：
├── Notion（知識庫、SOP、客戶案例）
├── Google Drive（財務、合約、內部文件）
├── Monday.com（項目進度、任務狀態）
├── GitHub（代碼、版本控制）
└── Slack（團隊決策、重要公告）
```

#### Step 1.3：生成驗證清單
```
通用驗證清單：
☐ 所有連結都能打開
☐ 所有數字都有來源
☐ 所有人名、角色都是最新的
☐ 所有時間戳都是準確的
☐ 所有流程都與實際操作一致
☐ 所有工具都是當前在用的
☐ 所有客戶案例都是真實的
☐ 沒有過時的信息（>3 個月未更新）
☐ 沒有矛盾的陳述
☐ 所有專業術語都是正確的
```

### Phase 2：執行驗證

#### Step 2.1：連結驗證
```
檢查項目：
□ 所有超連結都能訪問
□ 所有 Notion 連結都有正確的權限
□ 所有外部連結都還有效
□ 所有內部連結都指向正確的頁面
```

**工具**：使用自動化腳本檢查連結有效性

#### Step 2.2：數據驗證
```
檢查項目：
□ 所有數字都有來源（在文檔中標註或附件中提供）
□ 所有百分比都是準確的（計算無誤）
□ 所有時間都是準確的（日期、時間、時區）
□ 所有金額都是準確的（幣種、匯率、計算）
```

**方法**：逐一查詢原始數據源，確認一致性

#### Step 2.3：內容驗證
```
檢查項目：
□ 所有人名、角色都是最新的
□ 所有流程描述都與實際操作一致
□ 所有工具名稱都是正確的
□ 所有定義都是一致的
□ 沒有矛盾的陳述
```

**方法**：與 Dennis、團隊成員確認，或查詢最新的 Notion 文檔

#### Step 2.4：時效性驗證
```
檢查項目：
□ 文檔最後更新日期是什麼時候
□ 是否有過時的信息（>3 個月）
□ 是否有已停用的工具或流程
□ 是否有已離職的員工
```

**方法**：檢查文檔的更新日期，查詢全局知識地圖確認當前狀態

### Phase 3：記錄問題

#### Step 3.1：分類問題
```
問題分類：

P0（嚴重）：
- 連結無法訪問
- 數據嚴重錯誤（差異 >10%）
- 人名、角色錯誤
- 已停用的工具或流程

P1（中等）：
- 連結需要更新
- 數據略有誤差（差異 <10%）
- 信息不夠清晰
- 缺少來源標註

P2（輕微）：
- 排版、格式問題
- 表述不夠精確
- 可以優化的地方
```

#### Step 3.2：記錄問題
```
問題記錄格式：
- 問題位置：第 X 頁，第 Y 段
- 問題描述：具體是什麼問題
- 優先級：P0/P1/P2
- 建議修復：如何修復
- 數據來源：從哪裡查到的正確信息
```

### Phase 4：改善與驗證

#### Step 4.1：修復問題
```
修復順序：
1. 先修復 P0 問題（必須修復）
2. 再修復 P1 問題（應該修復）
3. 最後優化 P2 問題（可以優化）
```

#### Step 4.2：複查
```
複查項目：
□ 所有 P0 問題都已修復
□ 所有 P1 問題都已修復
□ 修復後沒有引入新問題
□ 文檔的邏輯還是清晰的
```

#### Step 4.3：發布
```
發布前檢查：
□ 所有問題都已解決
□ 文檔已更新版本號
□ 文檔已標註最後更新日期
□ 文檔已標註維護者
□ 文檔已備份（Google Drive、Notion、GitHub）
```

```python
from urllib.parse import urlparse

def verify_links(content):
    """驗證文檔中的所有連結"""
    links = extract_links(content)
    results = []
    
    for link in links:
        try:
            response = requests.head(link, timeout=5)
            status = "✅ 可訪問" if response.status_code == 200 else f"❌ 狀態碼 {response.status_code}"
        except:
            status = "❌ 無法訪問"
        
        results.append({
            "link": link,
            "status": status
        })
    
    return results
```

## 💡 最佳實踐

### 1. 建立「單一真實來源」
- 所有信息都應該有明確的來源
- 當多個來源衝突時，以最新的為準
- 定期更新數據源，確保信息最新

### 2. 自動化優先
- 使用腳本自動檢查連結、數據格式
- 減少手動檢查的工作量
- 提高檢查的一致性

### 3. 定期複查
- 新發布的內容每週複查一次
- 已發布的內容每月複查一次
- 超過 3 個月未更新的內容標記為「需要複查」

### 4. 文檔化所有決策
- 當發現問題並修復時，記錄下來
- 建立「常見錯誤」清單
- 幫助未來避免同樣的問題

## ⚠️ 常見陷阱

### 陷阱 1：過度依賴記憶

❌ 「這個工具應該還在用」

### 陷阱 4：只驗證一次
