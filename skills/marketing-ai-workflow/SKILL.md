# 好創 AI 內容生產工作流

本工作流供好創任何團隊成員使用。執行前請同步載入 `haochuang-brand-context` skill。

## 好創的內容服務邊界

**承接：** 短影音腳本（Reels/TikTok/Shorts）、SEO/AIO/GEO 文章、Meta 廣告文案、Google 廣告文案、社群貼文、電子報、服務頁面文案、品牌故事。

**不承接（或非主力）：** 高階形象影片腳本（非主力，必要時外包）、單點文案接案（必須打包進整合服務）、非繁中台灣市場的內容。

## 好創的客戶類型與限制

| 客戶類型 | 常見痛點 | 內容切入角度 |
|---------|---------|------------|
| 高雄中小企業主 | 預算有限、不懂數位行銷 | 投報率、省時省力、本地案例 |
| 電商品牌 | 流量貴、轉換低 | 精準受眾、再行銷、數據說話 |
| 服務業（餐飲/診所/補習班） | 口碑難量化、客源不穩 | 社群信任感、在地 SEO |

**溝通限制：** 所有內容不得出現未經驗證的數據、不得承諾具體排名或流量數字、不得使用「通常」「大家都知道」等模糊詞。

## 任務判斷

```
收到內容任務 →
  是否為短影音腳本？ → 是：必須同步載入 reels-script-writer 技能執行
  是否為好創自有內容？→ 是：語氣第一人稱「好創」，存入 haochuang-content repo
  是否為客戶代操？   → 是：確認客戶品牌語氣，存入客戶專屬 repo
  是否需要批量（5篇+）？→ 是：先確認關鍵字清單與受眾，再批量生成
```

## 交付標準（每次都要執行）

1. **格式：** Markdown，含 frontmatter（title、meta_description、target_keyword、word_count）
2. **品質審查：** 事實正確（無幻覺）、關鍵字自然出現、CTA 明確、語氣符合好創規範
3. **版控：** 完成後存入對應 GitHub Repository，commit message 格式：`content: [類型] [主題關鍵字]`
4. **通知：** 發布完成後發送 Slack 通知到 #content 頻道，格式：`[類型] 《標題》 已完成，連結：[GitHub URL]`
5. **排程：** 發布日期記錄在 Monday.com 對應項目的「內容日曆」欄位

## 提示詞模板

使用前讀取 `templates/prompts.md`，包含 SEO 文章、廣告文案、社群貼文三種模板。
