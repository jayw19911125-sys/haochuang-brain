# Session State Exporter (對話狀態打包器)

本技能專門用於將當前對話的工作狀態，進行「高密度壓縮」與「去蕪存菁」，並輸出成緊湊的代碼區塊（Code Block），讓使用者可以一鍵複製並貼到新的對話中，實現 **100% 狀態無損轉移**。

⚠️ **警告：這不是學習筆記！**
- 學習筆記（learning-notes-generator）是為了沉澱知識。
- 本技能（session-state-exporter）是為了「記憶轉移與接續工作」。

## 執行流程 (Execution Flow)

當使用者觸發此技能時，請嚴格按照以下步驟執行：

### Step 1: 盤點全局對話狀態 (State Audit)
掃描整個對話歷史，過濾掉寒暄、情緒發言、已解決且不影響後續的錯誤嘗試。精煉出以下五個核心要素：
1. **TASK_STATE (當前卡點)**：目前確切停在哪個步驟？正在等待什麼？
2. **DONE (已完成)**：已經確認完成且不可推翻的決策或進度。
3. **TODO (待辦事項)**：原本預計要做但還沒做的事情。
4. **CONSTRAINTS (限制條件)**：對話中確認過的鐵則（例如：預算上限、必須使用特定工具、禁止使用的詞彙）。
5. **FORGOTTEN_RISKS (盲點與遺漏)**：對話中曾經提過，但被擱置或尚未處理的潛在風險。

### Step 2: 高密度壓縮 (High-Density Compression)
將上述五個要素壓縮至極致，**必須比原始對話縮小 50% 以上**。
- 移除所有形容詞、副詞與過渡句。
- 僅保留動詞、名詞與關鍵數據。
- 採用條列式或 JSON 結構。

### Step 3: 輸出打包代碼 (Output Export Block)
將壓縮後的狀態，包裹在一個 `json` 的代碼區塊中，並附上一段給「下一個 AI」的喚醒指令。

## 輸出格式模板 (Output Template)

請直接將以下內容（替換為實際數據後）輸出給使用者，**不要附帶任何多餘的解釋或廢話**。

```markdown
請複製以下區塊，貼到新的對話中：

\`\`\`json
{
  "__WAKE_UP_COMMAND__": "讀取以下狀態，這是一個從舊對話轉移過來的工作進度。請直接回答『狀態已同步，請指示下一步』，不要做多餘的總結。",
  "TASK_STATE": "簡短描述當前停滯點",
  "DONE": ["動作1", "動作2"],
  "TODO": ["待辦1", "待辦2"],
  "CONSTRAINTS": ["限制1", "限制2"],
  "CONTEXT_SUMMARY": "濃縮核心背景（50字內）",
  "FORGOTTEN_RISKS": ["未解風險1"]
}
\`\`\`
```

## 執行鐵則 (Constraints)
1. **絕對精簡**：輸出的 JSON 區塊必須極度緊湊。
2. **無損轉移**：確保下一個 AI 讀到這段文字時，能立刻知道「現在要做什麼」以及「絕對不能做什麼」。
3. **零廢話交付**：使用者要求打包時，直接給出代碼區塊，不要說「我已經幫您打包好了...」。
