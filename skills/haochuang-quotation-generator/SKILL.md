# 好創報價單生成器 (Haochuang Quotation Generator)

此技能供好創團隊成員使用，旨在解決報價單無版本控管、紀錄分散的問題，並落實「現有 4 人核心團隊（CEO 陳涵勻、COO 黃子權、剪輯師阿韋、企劃小鑫）」的產能防禦機制與「先收錢才開工」的收款鐵則。

> 定價與收款比例尚在確認中，本技能**不寫死任何金額或百分比**。所有單價與付款節點，以使用者提供或公司大腦（`haochuang_brain.md`）確認的數字為準；資料不足時中斷提問，嚴禁自行假設。

## 觸發條件

使用者在對話中說出以下任何一句即觸發：
- 「報價」
- 「估價」
- 「幫我算一下這個案子多少錢」
- 「開個報價單」

## 核心規則（不可違背）

1. **產能防禦機制**：所有報價排程與產能計算，強制以「現有 4 人核心團隊（CEO 陳涵勻、COO 黃子權、剪輯師阿韋、企劃小鑫）」為底層邏輯。招募中職缺（剪輯師、行政、專案管理、短影音操盤手、AI 影音導演）在實際到職前不得計入產能。若專案規模超出此承載力，必須在報價中明確列出外包預算（如：瓦吉負責 SEO/廣告，起吉負責 KOL/公關）。
2. **收款鐵則**（先收錢才開工）：
   - 網站專案、社群類專案、混合專案的訂金比例、付款時點與拆分方式，**一律以公司大腦最新定價規則為準**。
   - 混合專案需按項目拆分，並在報價單中明確列出各項目的付款節點。
   - 若尚未確認具體比例，中斷並向使用者確認，不自行填入數字。
3. **單一真實來源**：報價單生成後，必須將關鍵資訊記錄至 Google Sheets（過渡期）或 Monday.com（最終目標），以利後續合約與專案啟動的串聯。

## 執行工作流

### 步驟一：需求釐清與參數收集

若使用者提供的資訊不足，系統必須中斷並直接提問，要求補充以下資訊（嚴禁自行假設）：
1. **客戶名稱與專案名稱**
2. **核心服務項目**（網站架設 / SEO / 短影音 / 商業攝影 / 廣告投放 / 品牌視覺 / 社群行銷）
3. **預期走期**（如：單月、三個月、半年）
4. **特殊需求或產能評估**（是否需要外包支援）

### 步驟二：報價計算與結構化產出

根據收集到的資訊，產出 Markdown 格式的結構化報價單。報價單必須包含以下區塊：

1. **專案摘要**：客戶名稱、專案名稱、報價日期、有效期限。
2. **服務項目與定價**：條列各項服務、單價、數量與小計。
3. **外包預算（若有）**：明確列出非編制人員的費用預估。
4. **付款條件與節點**：嚴格套用好創的「收款鐵則」（具體訂金比例以公司大腦或使用者確認為準，不自行填入）。
5. **產能影響評估**：簡述此專案對核心團隊（現有 4 人）的產能佔用情況。

### 步驟三：資料記錄與後續串聯

1. 將報價單 Markdown 檔案提供給團隊成員下載。
2. 將報價核心數據（客戶、金額、日期、狀態：待簽約）整理成可寫入 Google Sheets 或 Monday.com 的格式，並詢問是否要立即寫入系統。
3. 提示後續流程：「當款項入帳後，系統將自動啟動 `haochuang-client-onboarding` 流程，建立專案板並發送 Slack 通知。」

## 品質標準

1. **白話與結構化**：報價單內容必須白話易懂，並使用表格呈現費用明細。
2. **無幻覺**：所有金額與服務項目必須基於使用者提供或公司大腦確認的資訊，嚴禁自行腦補或套用未確認的定價。
3. **防禦性**：必須主動評估產能，避免接下無法負荷的專案。
