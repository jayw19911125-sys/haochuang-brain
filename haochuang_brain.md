## 抽屜 5：事情怎麼跑（流程與 SOP）

### 5.1 核心流程：報價 → 合約 → 收款 → 開票 → 交付（建置中，OKR-02）

1. 接觸客戶 → 業務／管理層洽談需求。
2. 填寫報價單（Netlify 報價單工具，含「案源」欄位：好創自開發 / 瓦吉帶入 / 起吉帶入）。
3. 確認送出 → 自動寫入 Google Sheets + Monday.com 建項目（狀態 `🔒 待付款`）。
4. 等待付款 → 專案凍結，不派工。
5. 款項入帳 → Monday 狀態改 `✅ 已付款 / 已收款`。
6. 解鎖自動派工 → 依「動態分流規則」建子任務、分配負責人、Slack 通知。

## 抽屜 9：AI 使用規範（新增）

### 9.1 哪些資料不得貼入 AI 工具

| 類型 | 範例 | 原因 |
| :--- | :--- | :--- |
| 客戶個資 | 姓名、電話、LINE ID | 個資法風險 |
| 合約內容 | 報價單、服務合約 | 商業機密 |
| 財務數字 | 毛利率、薪資、帳戶資訊 | 商業機密 |
| 未公開策略 | 競品分析、定價策略 | 競爭風險 |

### 9.2 AI 工具使用分級

| 等級 | 適用工具 | 可處理資料 |
| :--- | :--- | :--- |
| A 級（完整授權） | Manus（已接公司大腦） | 所有公司資訊 |
| B 級（一般使用） | ChatGPT、Claude、Gemini | 去除個資與財務數字後的資訊 |
| C 級（禁止） | 不明來源 AI 工具 | 任何公司資訊均不得使用 |

### 9.3 AI 大腦更新流程

1. 公司現況有任何變動（人員、定價、服務、OKR）。
2. 告訴 Manus「更新公司大腦」。
3. Manus 修改本檔並推送 GitHub → 網址內容立刻更新。
4. 其他 AI 工具下次讀取時自動獲得最新版。
